수요 공급

1. US : User Experience
> 사용자 경험
> 예) 알리익스프레스 A 불편한점 // 올리브영 B 불편한점 // 마켓컬리 C 좋은경험

2. UI : User Interface
> 사용자경험을 기반으로 더 좋은 디자인을 고안하고, 제공해주기 위한 결과물(디자인)

3. 왜 UX/UI 에 그렇게 열광하는 것일까?

4. 웹 > 모바일
- 젊은 고객 : 트랜디한 디자인 요구(변화의 속도가 매우 빠름) && 개발
- 마케팅 관점 : 독일 > 동일한 메세지를 일관성 반복적으로 전달
- 일관성있는 메세지 중요 (미디어 홍수 -> 충성고객 -> 돈)

5. 기업들은 UX/UI 기획 전문가가 필요
- UI 디자이너(UI 포커스)

6. UX/UI의 개발과정을 설명
- Double Diamond 이론
1) Discover : 현, 서비스의 문제점을 발견하는 단계
2) Define : 현, 서비스의 문제점들 중 우선순위를 결정하는 단계
3) Develop : 우리가 우선문제라고 판단한 문제들을 해결하기 위해 가설을 설정하는 단계
4) Deliver : 설정된 그 가설을 기반으로 UI 디자인을 고안 > 실제 유저를 대상으로 테스트하는 단계

7. 대상
- 위에서 우리가 설정한 그 문제점들은 과연 누구를 대상으로 선정한걸까?
- 그 사람들이 지금의 문제점들을 보다 효율적인 방법으로 수용할 수 있도록 가설
- 그 사람들을 대상으로 테스트
- 지금까지 우리가 논의해온 문제점을 경험, 개선, 테스트 => 대상 Target 고객

8. Target 고객은 어떻게 도출되는가?
- 잠재고객 : 현재는 우리의 고객이 아니지만, 우리의 고객으로 만들고 싶은 고객

9. 그렇다면 잠재고객은 어떻게 정의?
- 기업 혹은 브랜드의 비즈니스 목표에서 나옴
ex) 사장 : 목표(연매출 10억, 삼성 글로벌 시장 진출, 현기차 안정적인 시장 확보, 올리브영 폐점수 제로 관리, 올리브영 일본 진출 => 잠재 고객 일본MZ)

10. 올리브영 목표 > 잠재고객 > 일본MZ > 19세 ~ 28세(10여년의 나이차) -> 세대가 다름

11. 잠재고객(러프한 카테고리) > 세그먼트(segment)
- 세그먼트 = 그룹
- 19~23세, 오사카, 하라주쿠 => 역동적인 MZ
- 24~26세, 자신을 가꾸는데 돈을 아끼지 않는 실용파 MZ
- 27~28tp, 시노쿠보 거니는 경제적인 여유가 있는 MZ

12. 잠재고객 > 세그먼트 > 페르소나 > 연극 캐릭터를 지칭 용어 > 마케팅 
피그마 > 플러그인 > user profile avatar

13. 지금까지 도출된 잠재고객 > 세그먼트 > 페르소나들에게 설문조사 
=> User Research(*단계를 얼마나 고도화된 테크닉, UI 상관관계)

14. User Research 2가지의 분류
- Primary Research : 직접 페르소나를 대상으로 리서치 조사
- Secondary Research : 직접 페르소나를 만나러 가지는 않지만, 통계청 등의 외부 조사기관 혹은 웹 사이트를 통해서 간접적인 정보를 조사
