특정 이미지를 가져와서 원하는 사이즈 만큼 캡쳐한 후 원하는 위치에 해당 이미지를 출력
(이미지, 캡쳐하고자 하는 x좌표, 캡쳐하고자하는 y좌표, 너비, 높이, 출력하고자 하는 x좌표, 출력하고자하는 y좌표, 너비, 높이)


- innerHeight : 실제 개발자가 작성하는 문서가 출력되어지는 높이 공간
- innerWidth : 실제 개발자가 작성하는 문서가 출력되어지는 너비 공간
- outerHeight : 문서가 출력되어지는 공간 밖에 즐겨찾기 및 주소창 등의 공간까지 포함한 높이
- outerWidth : 우측 스크롤바의 공간까지 포함시킨 너비 공간
- pageXOffset : 우리가 작성한 문서를 기준으로 얼만큼 x축으로 떨어졌는지에 대한 좌표값
- pageYOffset : 우리가 작성한 문서를 기준으로 얼만큼 y축으로 떨어졌는지에 대한 좌표값
- screenX : 사용자의 모니터를 기준으로 x좌표값
- screenY : 사용자의 모니터를 기준으로 y좌표값

- clientX : 사용자가 바라보고 있는 매체=브라운저 화면의 전체너비 가운데 x좌표값
- clientY : 사용자가 바라보고 있는 매체=브라운저 화면의 전체너비 가운데 y좌표값
> 내부에 있는 문서의 너비 혹은 높이와는 전혀 무관 // 영향!!!
- offsetX : 현재 이벤트가 발생된 해당영역을 기준으로 얼만큼 x축으로 떨어져있는지에 대한 값
- offsetY : 현재 이벤트가 발생된 해당영역을 기준으로 얼만큼 y축으로 떨어져있는지에 대한 값
> 이벤트가 발생된 영역 안에서 값을 서칭
- pageX : 문서=html 문서상에서의 전체너비 가운데 x좌표값
- pageY : 문서상에서의 전체너비 가운데 y좌표값
> 스크롤까지 포함한 좌표값을 찾아온다
- screenX : 현재 사용자가 사용중인 모니터 화면을 기준으로 x좌표값
- screenY : 현재 사용자가 사용중인 모니터 화면을 기준으로 y좌표값

* module
1. export라는 명령어를 활용해서 js 데이터를 출력하는 곳에서는 2가지 선택지가 존재함
   - export "default" : 1개의 자바스크립트 안에서 1개의 단일 함수 혹은 변수 혹은 객체를 대표해서 출력하는 경우
   - export : 1개의 자바스크립트 파일 안에 여러개 다수의 유틸리티용 함수 기능 탑재되어있는 경우
        => JS => 복수 함수(*상황에 따라 가져다가 사용)

2. import를 해야하는 자바스크립트 파일에서 2가지 방식으로 대응
    - 특정 변수 안에 고정된 객체의 값으로 가져오겠다는 의미
    - export : 해당 변수값으로 바로 찾아서 사용
